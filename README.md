I have learned the basic flow of git as a distributed version control system: first a change is made, then it is added to the staging area, then it is commited to the branch, then it is pushed on to the main branch
git skills I have learned:
git clone - adds a repo to a directory on my laptop
git add . - adds all changes in this directory to the staging area
git status - informs of what is in the staging area or not
git commit -m message - commits change to the local branch I working on, should be alongside a descriptive yet concise message 
git log - shows all the commits and messages I have made so far
git push - moves my changes on to the main branch of the repository
